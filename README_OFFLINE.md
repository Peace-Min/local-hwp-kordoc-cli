# Offline Installation

Use this flow when the target PC cannot access the internet.

## Clone-And-Run Install

This repository is prepared so the offline PC can install with one command after
`git clone`:

```bat
install-offline.cmd
```

That command:

- reassembles the Node.js MSI from `parts\`
- verifies the Node.js MSI SHA256
- installs Node.js if Node 18+ is not already available
- verifies the bundled `node_modules` runtime dependencies
- runs the CLI smoke check

If you only want to rebuild the Node.js installer, run:

```bat
reassemble-node.cmd
```

## What The Bundle Contains

The release-style offline bundle generated by
`scripts/prepare-offline-bundle.ps1` contains:

- this CLI application
- vendored kordoc parser runtime
- `node_modules` runtime dependencies
- an optional Node.js Windows MSI installer
- `install.ps1`
- `hwp-kordoc.cmd`

It does not contain Hancom Office, an HWP viewer, or any company DRM client.
Those must already be installed and authorized on the target PC when parsing
DRM/distribution documents.

## Build The Offline Bundle On An Internet PC

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\prepare-offline-bundle.ps1
```

The output ZIP is created under:

```text
dist\offline\
```

To skip downloading Node.js:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\prepare-offline-bundle.ps1 -SkipNodeDownload
```

To pin a specific Node.js MSI version:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\prepare-offline-bundle.ps1 -NodeVersion 22.14.0
```

## Install On The Offline PC

For this Git repository, prefer `install-offline.cmd` from the repository root.

For the generated ZIP bundle, extract the ZIP, then run:

```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

If Node.js is not already installed and the bundle includes a Node MSI, run:

```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1 -InstallNode
```

By default, the app is copied to:

```text
%LOCALAPPDATA%\Programs\hwp-kordoc-cli
```

Use a custom path:

```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1 -InstallDir D:\Tools\hwp-kordoc-cli
```

## Run

After install:

```powershell
hwp-kordoc.cmd .\sample.hwpx -o .\sample.md
hwp-kordoc.cmd .\sample.hwp --format json -o .\sample.json
```

If the install directory is not on `PATH`, run the command directly:

```powershell
%LOCALAPPDATA%\Programs\hwp-kordoc-cli\hwp-kordoc.cmd .\sample.hwpx -o .\sample.md
```

## Verify

```powershell
hwp-kordoc.cmd --version
hwp-kordoc.cmd --help
```
